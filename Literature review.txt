2.3 Problem Domain Review (~4 pages)
Establish the theoretical and contextual foundations of your topic.
2.3.1 Introduction to the Domain
Start broad: introduce the overarching research domain (e.g., cybersecurity, medical imaging, XAI, anomaly detection).
Define key terms and concepts using authoritative sources.
Keep it concise but clear (≈1 page).
2.3.2 Key Challenges / Sub-areas
Organise the problem domain into thematic sub-areas, typically 3–5. For each:
Define the concept clearly.
Explain why it is important.
Support with recent citations.
End with a linking sentence to the next sub-area.
Example sub-areas:
Data heterogeneity
Interpretability and transparency
Domain-specific constraints (e.g., real-time performance)
Adversarial robustness

2.4 Technological Review (~4 pages)
Critically examine relevant technologies stage by stage, not just the final algorithm. Example structure for ML projects:
Data Acquisition & Pre-processing (e.g., normalization, augmentation, anonymization)
Model Architectures (e.g., CNNs, Transformers, Federated frameworks)
Hyperparameter Tuning approaches
Evaluation Metrics used in the field
For non-ML projects (e.g., security), this could involve protocol standards, cryptographic primitives, system architectures, or forensic toolchains.
📌 Tip: Use sub-sections for each stage, and focus on critical analysis, not summaries.

2.5 Existing Work (~4 pages)
Use thematic analysis instead of one-paper-per-paragraph summaries.
2.5.1 Thematic Grouping
Identify major themes in the literature (e.g., methodological innovations, application domains, evaluation techniques).
Discuss multiple works together under each theme, highlighting trends and divergences.
2.5.2 Cross-theme Linking
Acknowledge overlapping works between themes.
Use linking sentences to maintain logical flow.
📌 Minimum 1 citation per sentence.
